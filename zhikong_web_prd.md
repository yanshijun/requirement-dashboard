# 智控中心 Web版 产品需求文档

**版本**：v1.1
**日期**：2026-03-12
**作者**：产品团队

---

## 一、背景与目标

智控中心目前集成在 Windows 客户端内，用户使用受设备限制。本次目标是将智控中心抽离为独立 Web 网页版，支持两种访问路径：

1. **从 Windows 应用跳转**：免登录，携带 token 直接进入
2. **独立域名直接访问**：需要登录，账号体系与 Windows 应用一致

---

## 二、用户角色与权限

| 角色 | 可访问模块 |
|------|------|
| 主账号 | 全部模块（含团队管理） |
| 子账号 | 除团队管理外的所有模块 |

> 子账号访问团队管理时，展示「暂无权限」提示页，不展示任何数据。子账号左侧导航不显示团队管理菜单项。

---

## 三、访问路径一：从 Windows 应用跳转

### 3.1 入口位置

不单独新增按钮，集成在现有交互中，避免界面冗余：

**方案一（推荐）**：用户点击右上角头像 / 账号名，下拉菜单中新增「在浏览器中打开」选项，点击后跳转

**方案二（备选）**：智控中心模块标题栏右侧放置小「外链」图标，hover 显示 tooltip「在浏览器中打开」，点击跳转

### 3.2 跳转行为

- 在系统默认浏览器中打开 Web 版智控中心
- 自动携带当前登录态（token），用户无需重新登录，无感知
- 打开后定位到与 Windows 当前所在模块相同的页面
  - 示例：Windows 当前在「AI问答库」→ 浏览器打开后也在「AI问答库」

### 3.3 对 Windows 应用的影响

- 跳转行为不影响 Windows 应用的已登录状态
- Windows 应用继续正常使用，两端互不干扰

---

## 四、访问路径二：独立域名直接访问

### 4.1 访问方式

用户直接在浏览器输入域名访问 Web 版，需要登录后使用。

### 4.2 登录规则

- 账号密码与 Windows 应用账号体系完全一致，不新增账号
- 登录成功后进入 Web 版首页（数据看板）
- 登录态仅在浏览器侧维护，**不影响 Windows 应用已登录的状态**
- 支持「记住登录状态」，避免频繁重新登录

### 4.3 登录页设计要求

- 页面元素：产品 Logo、产品名称、账号输入框、密码输入框、登录按钮
- 支持回车键提交
- 登录失败给出明确错误提示（账号不存在 / 密码错误）
- 登录成功后回跳至目标页（若携带了目标 URL 参数则跳目标页，否则进首页）

---

## 五、Web 端页面结构

### 5.1 整体布局

```
┌─────────────────────────────────────────────┐
│  顶部栏：Logo + 产品名 + 用户信息 + 退出    │
├──────────┬──────────────────────────────────┤
│          │                                  │
│  左侧    │         主内容区域               │
│  导航栏  │    （与 Windows 版内容一致）     │
│          │                                  │
└──────────┴──────────────────────────────────┘
```

### 5.2 左侧导航模块

与 Windows 版完全对应，顺序一致：

1. 数据看板
2. 店铺库
3. 商品库
4. AI问答库
5. AI提问
6. 历史消息
7. 尺码规格
8. 自定义变量
9. 团队管理（主账号可见；子账号不显示此菜单项）

### 5.3 导航行为

- 进入页面时高亮当前模块（由 URL 参数决定）
- 切换模块时 URL 同步变化，支持浏览器前进 / 后退
- 刷新页面保持当前模块，不跳回首页

---

## 六、双版本共存规范

| 规则 | 说明 |
|------|------|
| 数据同步 | Windows 和 Web 操作同一份数据，任意一端修改另一端刷新后生效 |
| 功能对等 | Web 版功能与 Windows 版保持一致，不裁剪、不新增 |
| 登录互不干扰 | Web 端登录 / 退出不影响 Windows 端的登录状态 |
| 无主次之分 | 两个版本平行存在，均为正式版本 |

---

## 七、异常场景处理

| 场景 | 处理方式 |
|------|------|
| 子账号访问团队管理 | 展示「暂无权限」提示页，导航不显示该菜单项 |
| token 失效（路径一） | 跳转登录页，登录后回到原模块 |
| 登录态过期（路径二） | 跳转登录页，登录后回到原页面 |
| 网络异常 | 内容区展示加载失败提示，支持手动重试 |
| URL 模块参数不存在 | 跳转至数据看板（默认首页） |
| 登录失败 | 输入框下方展示错误原因，不清空已输入内容 |

---

## 八、验收标准

**路径一（Windows 跳转）**
- [ ] Windows 客户端账号菜单中可见「在浏览器中打开」入口
- [ ] 点击后在默认浏览器打开，自动登录无需手动输入账号密码
- [ ] 打开后定位到与 Windows 当前相同的模块
- [ ] 跳转后 Windows 应用登录状态不受影响

**路径二（独立访问）**
- [ ] 直接访问域名，未登录时跳转登录页
- [ ] 登录账号与 Windows 应用账号一致
- [ ] 登录成功后进入首页或目标模块
- [ ] 浏览器端登录 / 退出不影响 Windows 端状态

**通用**
- [ ] 主账号可见全部 9 个模块，子账号不显示团队管理
- [ ] URL 路由与模块一一对应，支持直接链接访问
- [ ] 双端数据一致，无数据孤岛
