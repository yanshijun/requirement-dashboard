# 飞书多维表格双向同步项目 - 技术架构与配置指南

> 本文档面向开发者，详细说明如何从零搭建一个与飞书多维表格双向同步的 Web 管理系统。
> 不含具体业务代码，只讲架构、配置、步骤。

---

## 一、项目架构总览

```
┌──────────────────────────────────────────────────┐
│           浏览器（前端 SPA 单页应用）               │
│         Vanilla JS / Vue / React 均可             │
└────────────────────────┬─────────────────────────┘
                         │  HTTP 请求 /api/*
                         ▼
┌──────────────────────────────────────────────────┐
│              中间层（Node.js 后端）                │
│                                                  │
│  职责：                                           │
│  ① 代理飞书 API（前端不直接调飞书）                 │
│  ② 管理 Token（获取、缓存、自动刷新）              │
│  ③ 字段格式转换（飞书格式 ↔ 前端格式）             │
│  ④ 连接 MySQL 存储本地扩展数据                    │
│  ⑤ 文件上传中转（base64 → 飞书云文档）            │
└──────┬───────────────────────┬───────────────────┘
       │                       │
       ▼                       ▼
┌──────────────┐       ┌──────────────┐
│  飞书开放平台  │       │    MySQL     │
│  (数据主源)   │       │  (扩展存储)   │
│              │       │              │
│ · 多维表格    │       │ · 向量数据    │
│ · 云文档存储  │       │ · 聊天记录    │
│ · 附件管理    │       │ · 本地业务表  │
└──────────────┘       └──────────────┘
```

### 核心原则

| 原则 | 说明 |
|------|------|
| 飞书是数据主源 | 所有业务数据以飞书多维表格为准，前端每次加载从飞书拉取最新 |
| 后端做中转 | 前端不直接调飞书 API，避免暴露 App Secret |
| 双向同步 | 前端修改 → 写回飞书；飞书直接改 → 刷新即可看到 |
| MySQL 做补充 | 飞书不适合存的数据（向量、大文本、高频读写）放 MySQL |

## 二、飞书开放平台配置（第一步）

### 2.1 创建应用

1. 打开 https://open.feishu.cn/app → 点击「创建企业自建应用」
2. 填写应用名称和描述 → 创建完成
3. 进入应用 → 「凭证与基础信息」页面，记录：
   - **App ID**：`cli_xxxxxxxxxx`（应用唯一标识）
   - **App Secret**：`xxxxxxxxxxxxxxxx`（应用密钥，不要泄露）

### 2.2 开通权限

进入应用 →「权限管理」→ 搜索并开通以下权限：

| 权限名称 | 权限标识 | 用途 |
|---------|---------|------|
| 查看、评论和编辑多维表格 | `bitable:app` | 读写多维表格数据 |
| 上传和下载文件 | `drive:drive` | 附件上传到飞书云文档 |

开通后需要管理员审批通过。

### 2.3 发布应用

权限开通后 → 「版本管理与发布」→ 创建版本 → 提交审核 → 管理员通过后应用生效。

### 2.4 创建多维表格

1. 在飞书中新建一个多维表格
2. 设计好字段（文本、单选、多选、日期、进度等）
3. 从浏览器地址栏获取两个关键 ID：

```
URL 格式：https://xxx.feishu.cn/base/XCgIb7NnwaAQq3sf1mjcQtdyned?table=tblgnpLLYI2JGQH0

                                       ↑ APP_TOKEN                    ↑ TABLE_ID
```

4. 将应用添加为多维表格的协作者（否则 API 无权访问）：
   - 打开多维表格 → 右上角「...」→「添加协作者」→ 搜索你的应用名 → 添加为「可编辑」

## 三、服务器环境配置（第二步）

### 3.1 Node.js 环境

```bash
# 安装 Node.js（建议 18+）
# 初始化项目
mkdir my-project && cd my-project
npm init -y

# 安装依赖
npm install express           # 本地开发服务器
npm install mysql2            # MySQL 连接
```

### 3.2 MySQL 数据库

准备一台 MySQL 服务器（本地或云服务器均可），记录：

| 配置项 | 示例值 | 说明 |
|--------|--------|------|
| DB_HOST | `8.135.12.29` | 数据库地址 |
| DB_PORT | `3306` | 端口 |
| DB_USER | `root` | 用户名 |
| DB_PASS | `your_password` | 密码 |
| DB_NAME | `my_database` | 数据库名 |

确保：
- 数据库字符集为 `utf8mb4`（支持中文和 emoji）
- 从你的服务器 IP 可以连接到数据库（防火墙/安全组放行）

### 3.3 环境变量

创建 `.env` 文件（不要提交到 Git）：

```env
# 飞书
FEISHU_APP_ID=cli_xxxxxxxxxx
FEISHU_SECRET=your_app_secret

# 数据库
DB_HOST=8.135.12.29
DB_PORT=3306
DB_USER=root
DB_PASS=your_password
DB_NAME=my_database

# AI（可选，如需智能问答）
QWEN_API_KEY=sk-xxxxxxxx
```

## 五、飞书 API 调用流程（第四步）

### 5.1 认证流程

```
应用启动
    │
    ▼
POST /auth/v3/tenant_access_token/internal
    │  请求体：{ app_id, app_secret }
    │  返回：{ tenant_access_token, expire }
    ▼
缓存 Token（内存中，设 90 分钟过期）
    │
    ▼
后续所有飞书 API 请求带上 Header：
    Authorization: Bearer {token}
    │
    ▼
如果返回错误码 99991663（token 过期）
    → 清除缓存 → 重新获取 → 重试请求
```

### 5.2 读取数据流程

```
前端请求 GET /api/feishu?table=xxx
    │
    ▼
后端获取 Token
    │
    ▼
GET /bitable/v1/apps/{APP_TOKEN}/tables/{TABLE_ID}/records?page_size=500
    │  （如果 has_more=true，用 page_token 继续翻页）
    ▼
拿到全部 records[]
    │
    ▼
遍历每条 record，调用 mapFromFeishu() 转换字段格式
    │  · 日期：毫秒时间戳 → "YYYY-MM-DD"
    │  · 进度：0.75 → 75
    │  · 多选：["A","B"] → "A,B"
    │  · 文本：[{text:"xxx"}] → "xxx"
    ▼
返回 JSON 数组给前端
```

### 5.3 写入数据流程

```
前端提交表单 POST /api/feishu?action=add
    │  请求体：{ name:"xxx", status:"开发中", ... }
    ▼
后端调用 mapToFeishu() 转换为飞书字段格式
    │  · "2026-05-01" → 1746028800000（毫秒时间戳）
    │  · 75 → 0.75
    │  · "A,B" → ["A","B"]
    ▼
POST /bitable/v1/apps/{APP_TOKEN}/tables/{TABLE_ID}/records
    │  请求体：{ fields: { "字段名": 值, ... } }
    ▼
飞书返回 record_id（新记录的唯一 ID）
    │
    ▼
返回给前端 → 前端更新本地数据 → 重新渲染
```

### 5.4 文件上传流程

```
用户选择/粘贴图片
    │
    ▼
前端压缩图片（canvas → JPEG, max 1920px）
    │
    ▼
转为 base64 字符串
    │
    ▼
POST /api/upload  { filename, mimetype, base64 }
    │
    ▼
后端解码 base64 → Buffer
    │
    ▼
构建 multipart/form-data 请求体
    │
    ▼
POST 飞书 /drive/v1/medias/upload_all
    │  · file_name: 文件名
    │  · parent_type: "bitable_file"
    │  · parent_node: APP_TOKEN（关联到哪个多维表格）
    │  · file: 二进制内容
    ▼
飞书返回 file_token（文件唯一标识）
    │
    ▼
前端保存 { file_token, name, type } 到记录的 attachments 字段
```

### 5.5 文件预览流程

```
用户点击附件
    │
    ▼
前端请求 POST /api/feishu?action=file-download  { file_token }
    │
    ▼
后端用 Token 请求 GET /drive/v1/medias/{file_token}/download
    │
    ▼
拿到二进制数据 → 转 base64 或直接流式返回
    │
    ▼
前端渲染：图片直接显示 / 其他文件提供下载链接
```

## 六、飞书字段类型对照表

开发时最容易踩坑的地方就是字段格式转换，以下是完整对照：

| 飞书字段类型 | 飞书存储格式 | 读取时转换 | 写入时转换 |
|-------------|-------------|-----------|-----------|
| 文本 | `[{type:"text", text:"内容"}]` | 取第一个元素的 `.text` | 直接传字符串即可 |
| 数字 | `123` | 直接用 | 直接传数字 |
| 单选 | `"选项A"` | 直接用 | 直接传字符串 |
| 多选 | `["选项A", "选项B"]` | `.join(",")` | `.split(",")` 转数组 |
| 日期 | `1746028800000`（毫秒时间戳） | `new Date(ts)` 格式化 | `new Date(str).getTime()` |
| 复选框 | `true` / `false` | 直接用 | 直接传布尔值 |
| 进度 | `0.75`（0~1 小数） | `× 100` 转百分比 | `÷ 100` 转小数 |
| 附件 | `[{file_token, name, ...}]` | 直接用 | 需先上传拿 token |
| 人员 | `[{id:"xxx", name:"张三"}]` | 取 `.name` | 需传 `[{id:"user_id"}]` |
| 超链接 | `{text:"显示文本", link:"URL"}` | 取 `.text` 或 `.link` | 传对象 |

## 七、本地开发服务器配置（第五步）

### 7.1 server.js 的作用

本地开发时用 Express 模拟 Netlify Functions 的运行环境：

```
浏览器请求 /api/feishu?action=list
       │
       ▼
Express 路由匹配 /api/feishu
       │
       ▼
将 Express 的 req 转换为 Netlify Function 的 event 格式：
  {
    httpMethod: req.method,
    queryStringParameters: req.query,
    body: JSON.stringify(req.body),
    headers: req.headers
  }
       │
       ▼
调用 handler(event) → 拿到 { statusCode, headers, body }
       │
       ▼
用 res.status().set().send() 返回给浏览器
```

### 7.2 启动方式

```bash
node server.js
# 输出：✅ 服务已启动: http://localhost:3000
```

修改代码后需要重启（或用 nodemon 自动重启）：
```bash
npm install -g nodemon
nodemon server.js
```

## 八、线上部署配置（第六步）

### 8.1 Netlify 部署

1. 将代码推送到 GitHub
2. 登录 https://app.netlify.com → 导入 GitHub 仓库
3. 构建设置：
   - Build command：留空（纯静态 + Functions）
   - Publish directory：`.`
4. 环境变量设置（Site settings → Environment variables）：
   - `FEISHU_SECRET`
   - `DB_HOST` / `DB_PORT` / `DB_USER` / `DB_PASS` / `DB_NAME`
   - `QWEN_API_KEY`（如需 AI）

### 8.2 netlify.toml 配置

```toml
[build]
  publish = "."

[functions]
  directory = "netlify/functions"
  node_bundler = "esbuild"

# 每个后端模块一条重定向规则
[[redirects]]
  from = "/api/feishu"
  to = "/.netlify/functions/feishu"
  status = 200

[[redirects]]
  from = "/api/upload"
  to = "/.netlify/functions/upload"
  status = 200

# 安全头
[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "SAMEORIGIN"
    X-Content-Type-Options = "nosniff"
```

### 8.3 数据库网络配置

确保 Netlify Functions 能连接到你的 MySQL：
- 如果用云数据库（阿里云/腾讯云）→ 安全组放行 `0.0.0.0/0`（或 Netlify IP 段）
- 如果用内网数据库 → 需要公网映射或 VPN

## 九、关键注意事项

### 飞书 API 限制

| 限制项 | 数值 | 应对方式 |
|--------|------|---------|
| 单次读取上限 | 500 条/页 | 用 page_token 循环翻页 |
| 批量写入上限 | 500 条/次 | 分批提交 |
| Token 有效期 | ~2 小时 | 缓存 90 分钟，过期自动刷新 |
| 请求频率 | 100 次/秒 | 批量操作加延时 |
| 文件大小 | 20MB | 图片压缩后上传 |

### 安全清单

| 项目 | 做法 |
|------|------|
| App Secret | 只存环境变量，不写代码里 |
| 数据库密码 | 只存环境变量 |
| .env 文件 | 加入 .gitignore |
| CORS | 生产环境限制为具体域名 |
| 文件上传 | 限制大小 + 类型校验 |
| SQL 注入 | 使用参数化查询（`?` 占位符） |

### 常见踩坑

| 问题 | 原因 | 解决 |
|------|------|------|
| API 返回 "no permission" | 应用未添加为表格协作者 | 多维表格 → 添加应用为协作者 |
| 日期显示 1970 年 | 飞书返回毫秒时间戳，误当秒处理 | 确认单位是毫秒 |
| 多选字段写入失败 | 传了字符串，应传数组 | `"A,B".split(",")` |
| 文件上传 400 | parent_node 不对 | 用目标多维表格的 APP_TOKEN |
| Token 过期 | 缓存时间太长 | 设 90 分钟过期 + 错误码重试 |
| 中文乱码 | 数据库字符集不对 | 建库时指定 `utf8mb4` |
| JSON 列返回对象不是字符串 | mysql2 自动解析 JSON 列 | 先判断 `Array.isArray()` |
